3계층 클라이언트/서버 구조

일반적인 클라이언트/서버 시스템 설계에서는 3계층형 구조를 사용한다.

1. 웹 클라이언트

웹 클라이언트(웹 브라우저)는 서버로 요청을 보내고, 서버가 보내온 응답을 화면에 출력하는 역할을 한다. 웹 상에서 클라이언트와 서버는 HTTP 프로토콜 이라는 약속된 규약으로 요청/응답한다. 브라우저는 주로 HTML 코드를 파싱하여 화면에 표시한다. (* 네트워크, 브라우저 동작 원리는 이후 포스팅 참고)

2. 웹 서버

웹 서버는 클라이언트의 요청을 받아 정적 페이지(HTML, CSS, 이미지 등)를 생성한다. 대표적인 예로 Apache, Nginx, IIS가 있다.

3. 웹 어플리케이션 서버 (WAS)

웹 서버와 웹 컨테이너의 결합으로 다양한 역할을 수행한다. 웹 컨테이너는 HTML만으로는 할 수 없는 데이터베이스 조회나 다양한 로직 처리와 같은 동적인 데이터 처리를 한다. 대표적으로 Tomcat, Jeus가 있다.

* 웹 서버와 WAS의 차이는?

웹 컨테이너의 기능(동적 컨텐츠 처리)의 유무가 가장 큰 차이이다.

* 일반적인 데이터 흐름은?

브라우저 <-> 웹 서버 <-> 웹 어플리케이션 서버 (WAS) <-> DB

* 그렇다면 WAS만 쓰면 되지 왜 웹 서버를 따로 쓰는가?

웹 서버와 WAS는 역할이 다르다. 단순한 HTML 파일이나 이미지는 웹 서버를 이용하는 것이 WAS를 이용하는 것보다 빠르고 안정적이다. 반면 DB 서버와 연결되어 데이터를 주고 받거나 변경이 있을 경우에는 WAS를 활용해야 한다.
