서버 구성 요소

( * 서버와 PC는 물리적으로 기본 구성이 같으므로 컴퓨터 구성 요소로 봐도 무방하다.)

CPU (Central Processing Unit)

서버 중심에 위치하여 연산을 처리한다. 사람의 두뇌에 해당하는 부분! 기억 장치나 입출력 장치를 통해 전달된 명령과 데이터를 받아, 연산을 실행하고 결과를 반환한다. 연산은 1초에 10억 회 이상 실행 가능하다.

  • 코어 : CPU의 핵심 부분
  • 클럭 : CPU의 속도 (단위: GHz)
  • 캐시 메모리 : CPU 안에 포함되는 빠르고 작고 비싼 메모리다. 처리 지연을 줄이기 위해 가장 자주 사용하는 명령/데이터를 코어와 가까운 곳에 배치한다. L1 캐시와 레지스터는 같은 개념이 아니다. L1 캐시에 있는 데이터도 궁극적으로 레지스터에 올라가야 CPU에서 처리할 수 있다. 레지스터는 CPU 아키텍처에 따라 종류와 의미가 다르다.
    • 데이터 지역성의 원리
      대부분 프로그램은 한 번 사용한 데이터를 다시 사용할 가능성이 높다. 그래서 메인 메모리에 있는 데이터를 캐시 메모리에 불러와 두고 CPU가 필요한 데이터를 캐시에서 먼저 찾도록 하여 시스템 성능을 향상시킬 수 있다.

CPU 하나가 처리할 수 있는 양에는 한계가 있다.
-> 여러 개의 CPU를 배치하면 처리량을 늘릴 수 있다. (코어 개수 증가: 멀티코어/병렬화)
-> But, 그 작업이 분담할 수 있는 처리가 아니라면 코어를 아무리 늘려도 효과가 없다.
-> 이때는 CPU 클럭 주파수를 올린다. (직렬 처리 속도 증가)
-> …

정리!

  1. 직렬 처리로 속도를 올리는 데는 한계가 있다.
  2. 병렬화를 통해 속도는 빨라지지 않지만, 단위 시간당 처리량을 늘릴 수 있다.
  3. 병렬 처리에서는 합류점, 직렬화 구간, 분기점이 병목 지점이 되기 쉽다. 따라서, 오버헤드를 감안하더라도 효과가 있을 경우에 병렬화 한다.

메모리

CPU 옆에 위치하는 기억 장치로, CPU에 전달하는 내용이나 데이터를 저장하거나 처리 결과를 받는다. 메모리에 저장되는 정보는 영구성이 없다. (휘발성) 따라서 서버를 재시작하면 데이터가 사라진다. 그럼에도 사용하는 이유는 메모리 액세스가 매우 빠르게 이루지기 때문이다. 데이터 저장 시, 물리적인 모트를 구동하는 것이 아니라 전기적인 처리만으로 데이터를 저장하기 때문이다. CPU 자체의 메모리(레지스터, L1 캐시, L2 캐시 등)와 다르다. CPU 내부에 있는 것이 아니다. 메모리 컨트롤러와 채널을 경유하여 CPU 밖으로 나가 메모리를 이용한다.

I/O 장치

하드 디스크 드라이브 (HDD)

장기 저장 목적의 데이터 저장 장소로, 디스크는 메모리와 달리 전기가 없어도 데이터가 사라지지 않는다. (비휘발성) 자기 원반이 고속으로 회전하여 읽기/쓰기 처리를 한다. CD나 DVD와 같은 구조. 이러한 회전 구조 때문에 소도가 물리 법칙에 좌우되며 메모리처럼 순식간에 액세스할 수 없다.

  • SSD (Solid State Disk, 반도체 디스크) : 최근 기술의 발달로 하드 디스크를 대체하기 위해 개발되었다. 자가 디스크인 HDD와 다르게 플래시 메모리로 이루어져 있다. 물리적인 회전 요소를 사용하지 않아 소음이 없고, 일반적으로 소모 전력과 발열도 적다. 데이터 입출력 속도가 HDD에 비해 매우 빠르며 작고 가볍다. HDD보다 가격이 비싸지만 현재는 가격이 하락하여 대중화되었고 점유율 50%를 넘겼다.

네트워크 인터페이스

서버와 외부 장비를 연결하기 위한 외부 접속용 인터페이스다.

I/O 제어

  • I/O 핸들러 (IOH) / 노스 브릿지 (north bridge)
    이전에는 메모리 I/O 제어가 주된 역할 이었으나, CPU에 그 역할이 옮겨가 지금은 다른 고속 처리가 필요한 I/O를 제어한다. 그리고 CPU 간 데이터 전송 제어도 한다.

  • I/O 컨트롤러 (ICH) / 사우스 브릿지 (south bridge)
    속도가 느려도 괜찮은 DVD나 USB 등의 I/O 제어를 담당하며, IOH 간 데이터 전송 제어도 한다.

버스

서버 내부에 있는 컴포넌트들을 서로 연결시키는 회선이다. 컴포넌트들은 버스로 연결된다.

  • 대역 : 데이터 전송 능력. 한 번에 데이터를 보낼 수 있는 데이터의 폭(전송폭), 1초에 전송할 수 있는 횟수(전송 횟수)로 결정된다. 버스 대역을 고려하여 CPU와 장치 사이에 병목 현상이나 낭비가 발생하지 않도록 설계하는 것이 중요하다.

전체 데이터의 흐름

CPU에서 가까울수록 고속이고 멀수록 대용량이다.
